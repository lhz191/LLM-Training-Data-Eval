│  ─────────────────────────────────────────────────────────────────────────── │

│                                                                              │
│  如果不能 → 这条数据有问题！                                                  │
│  ─────────────────────────────────────────────────────────────────────────── │

│  要么是原始页面本身就有歧义（需要人类也无法区分的情况）                         │
│                                                                              │

│    • 难度分布：负样本与 GT 的语义相似度分布                                    │
│    • 合理区间：太低（trivial）不好，太高（ambiguous）也不好                    │
│  意义：                                                                      │
│    • 太简单 → 模型学不到区分能力                                              │
│    • 太难 → 可能学到错误模式或无法收敛                         





核心观点：没有单一完美指标，需要多维度组合评估。而且最终的验证还是要看"训练后模型在真实任务上的表现"。

│  可区分性：关注的是"结果"                                                     │
│  ─────────────────────────────────────────────────────────────────────────── │
│  • 不管你怎么简化，最终 GT 能不能被识别？                                      │
│  • 只看简化后的数据                                                          │
│                                                                              │
│  信息密度：关注的是"过程/效率"                                                │
│  ─────────────────────────────────────────────────────────────────────────── │
│  • 你简化了多少？删掉的是不是都是垃圾？                                        │
│  • 需要对比简化前后                                                          │
│                                                                              │
│  关系：                                                                      │
│  ─────────────────────────────────────────────────────────────────────────── │
│  • 可区分性高 + 信息密度高 = 最好（删得多，但没删错）                          │
│  • 可区分性高 + 信息密度低 = 还行（没删多少，但至少没删错）                     │
│  • 可区分性低 + 信息密度高 = 有问题（删得多，但删错了）                         │
│  • 可区分性低 + 信息密度低 = 最差（没删多少，但还是删错了）                     │
